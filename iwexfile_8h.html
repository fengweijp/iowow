Detailed Description

Auto-expandable file.

Author
Anton Adamansky (adama.nosp@m.nsky.nosp@m.@gmai.nosp@m.l.co.nosp@m.m)
Note
Before using API of this module you should call iw_init(void) iowow module initialization routine.

Features:

  • Tuneable file expansion policies. Custom file resize policies supported by specifying IWFS_EXT_OPTS::rspolicy option value. The following policies are implemented:
    • Exact. File resizing fits exactly to the size required by write operation. T his is the default behaviour.
    • Fibonacci policy. Next file size computed accourding to fibonacci sequence of previous file sizes: file_size(n+1) = MAX(file_size(n) + file_size(n-1), nsize)
    • Multiplication resize policy. Next file size: file_size(n+1) = N * file_size(n) where N is a rational number IW_RNUM greater than 1
  • Read/write locking over a file's address space in multithreaded environment.
  • File shrinking/truncation support.
  • A number mmaped regions can be registered in the file's address space. These regions used in read/write operation and automatically maintained during file resize operations.

File operations implemented as function pointers contained in IWFS_EXT C structure. The iwrc iwfs_exfile_open(IWFS_EXT *f, const IWFS_EXT_OPTS *opts); opens file and initializes a given IWFS_EXT structure.

typedef off_t(* IW_EXT_RSPOLICY) (off_t nsize, off_t csize, struct IWFS_EXT *f, void **ctx)

File resize policy function type.

This function called in the following cases:

  • When a file needs to be resized. Returned new file size cannot be lesser than requested nsize and must be page aligned.
  • When a file is closed. In this case the first argument nsize will be set to -1 and function should return 0. This call can be used in order to release resources allocated for ctx private data used in function.
Parameters
nsizeDesired file size.
csizeCurrent file size.
fFile reference.
ctxFunction context data pointer. A function is allowed to initialize this pointer by oun private data stucture.
Returns
Computed new file size.

IW_EXPORT WUR iwrc iwfs_exfile_open ( IWFS_EXT *  f,
const IWFS_EXT_OPTS *  opts 
)

Open exfile.

Example:

Open a file for multithreaded env with fibonacci file resize policy and initial size to 4K

IWFS_EXT_OPTS opts = {
.file = {
.path = "myfile.dat",
.omode = IWFS_OWRITE | IWFS_OCREATE,
.lock_mode = IWP_WLOCK
},
.initial_size = 4096,
.use_locks = 1,
.rspolicy = iw_exfile_szpolicy_fibo
};
IWFS_EXT f;
iwrc rc = iwfs_exfile_open(&f, &opts);
rc = f.write(&f, ...);
...
