Detailed Description

Auto-expandable file with support of reader/writer address space locking and free space block management using bitmaps.

Author
Anton Adamansky (adama.nosp@m.nsky.nosp@m.@gmai.nosp@m.l.co.nosp@m.m)
Note
Before using API of this module you should call iw_init(void) iowow module initialization routine.

Features:

  • Address blocks allocation and deallocation using bitmaps.
  • Read/write file address space locking.
  • Tunable file expansion policies.
  • Read/write methods locking option in multithreaded environment.
  • File shrinking/truncation support.
  • A number mmaped regions can be registered in the file's address space. These regions used in read/write operation and automatically maintained during file resize operations.

File operations implemented as function pointers contained in IWFS_FSM C structure. The iwfs_fsmfile_open(IWFS_FSM *f, const IWFS_FSM_OPTS *opts) opens file and initializes a given IWFS_FSM structure.

File format: 

 [FSM_CTL_MAGICK u32][block pow u8]
 [bmoffset u64][bmlength u64]
 [crzsum u64][crznum u32][crszvar u64][reserved u256]
 [custom header size u32][custom header data...]
 [fsm data...]

where:

  • FSM_CTL_MAGICK: Free-space file magic number (32 bit)
  • block pow: Block size as power of 2 Eg: 6 means 64 bit block size. (8 bit)
  • bmoffset: Free space bitmap area offset in bytes (64 bit)
  • bmlength: Free space bitmap area length. (64 bit)
  • crzsum: Number of allocated blocks. (64 bit)
  • crznum: Number of all allocated continuous areas. (32 bit)
  • crszvar Allocated areas length standard variance (deviation^2 * N) (64 bit)
  • reserved: Reserved space.
  • custom header size: Length of custom header area. See IWFS_FSM::writehdr and IWFS_FSM::readhdr

IWFSM_ALLOC_NO_OVERALLOCATE 

Do not overallocate a requested free space in order to reduce fragmentation

IWFSM_ALLOC_NO_EXTEND 

Do not extend the file and its bitmap free space mapping in the case if file size expansion is required. In this case the IWFS_ERROR_NO_FREE_SPACE error will be raised.

IWFSM_ALLOC_PAGE_ALIGNED 

Force offset of an allocated space to be page aligned.

IWFSM_ALLOC_NO_STATS 

Do not update(collect) internal allocation stats for this allocation.

Open IWFS_FSM file.

Example:

Open a buffer pool file for multithreaded env with fibonacci file resize policy with block size of 64 bytes and custom file header of 255 bytes length.

IWFS_FSM_OPTS opts = {
.rwlfile = {
.exfile = {
.file = {
.path = "myfile.dat",
.omode = IWFS_OWRITE | IWFS_OCREATE,
.lock_mode = IWP_WLOCK
},
.rspolicy = iw_exfile_szpolicy_fibo
}
},
.bpow = 6, // 2^6 bytes block size
.hdrlen = 255, // Size of custom file header
.oflags = IWFSM_STRICT // Use verbose free-space bitmap checking for allocations (10-15% overhead)
};
IWFS_FSM f;
size_t sp;
off_t space_len, space_addr = 0;
iwrc rc = iwfs_fsmfile_open(&f, &opts);
//Allocate 2 blocks of file space
rc = f.allocate(&f, 128, &space_addr, &space_len, 0);
if (!rc) {
int data = 33;
//Write some data to the allocated block with writer lock acquired on `[space_addr, sizeof(data))`
rc = f.lwrite(&f, space_addr, &data, sizeof(data), &sp);
...
